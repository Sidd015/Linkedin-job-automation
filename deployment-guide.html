This application cannot run directly in a web browser because:

  • Browsers block direct SMTP connections for security
  • Email credentials cannot be safely stored in frontend code
  • You need a backend server to send emails

Solution: Deploy the backend API on a free hosting platform like Render or Railway (see instructions below).

üöÄ Deployment Options

1Option 1: Deploy on Render (Easiest - FREE)

Best for: Complete beginners, no technical knowledge required

  1. Create account at render.com
  2. Download the files I've created (they're in the outputs folder)
  3. Create a GitHub repository and upload: app.py, requirements.txt, Procfile, index.html
  4. In Render: Click "New +" ‚Üí "Web Service"
  5. Connect your GitHub repository
  6. Set:
    Build Command: pip install -r requirements.txt Start Command: gunicorn app:app
  7. Deploy! Copy the URL (e.g., https://your-app.onrender.com)
  8. Update index.html line 385: replace /api/send-emails with https://your-app.onrender.com/api/send-emails
  9. Deploy frontend as Static Site on Render

2Option 2: Deploy on Railway (Also FREE)

Best for: Slightly faster, auto-detects Python

  1. Create account at railway.app
  2. Click "New Project" ‚Üí "Deploy from GitHub repo"
  3. Railway auto-detects and deploys your Python app
  4. Click "Generate Domain" to get your URL
  5. Update frontend fetch URL with Railway URL

3Option 3: Run Locally (For Testing)

Best for: Testing before deploying online

# Install Python 3.8+ from python.org # Then run: pip install -r requirements.txt python app.py # Open index.html in your browser
